         Jamie Thornton, CUST-00421

## Example 3 — Multi-Agent (Delivery + Refund Simultaneously)

💬 **User types:**  
`"I want to track my delivery for order 99102 and also check my refund status for the mouldy cheese"`

🔍 **Pipeline trace:**
```
Stage 1: Not a greeting → continue
Stage 2: Load customer data:
         ORD-99102: In Transit | driver=Mike | stop=3/12
         ORD-97830: Delivered | Refund REF-77821 £14.99 → Processing | reason: Mouldy cheese
Stage 3: Guardrail → ALLOWED
Stage 4: No history → "new_retail"
Stage 5: "delivery" + "refund" in keywords → domain = "retail"
Stage 6: Skip DB (no nutrition keywords)
Stage 7: ⚠️ _get_direct_routing_tasks():
         → Message contains " and " → SKIP direct routing
         → Call _decompose_via_supervisor()

         Supervisor-Agent receives:
         "Decompose: 'I want to track my delivery for order 99102 and also
         check my refund status for the mouldy cheese'
         Available agents: order, delivery, refund, store"

         Supervisor-Agent returns:
         [
           {"agent": "delivery", "task_query": "Track delivery status for ORD-99102"},
           {"agent": "refund",   "task_query": "Check refund status for mouldy cheese in ORD-97830"}
         ]

Stage 8: ⚡ BOTH agents called IN PARALLEL (asyncio.gather)

         Delivery-Agent thread:
         [USER] context block with ORD-99102 data
         [USER] "Track delivery status for ORD-99102"
         [ASSISTANT] "Your order ORD-99102 is on stop 3 of 12..."

         Refund-Agent thread (simultaneously):
         [USER] context block with ORD-97830 refund data
         [USER] "Check refund status for mouldy cheese in ORD-97830"
         [ASSISTANT] "Your refund REF-77821 of £14.99 is currently being processed..."

Stage 9: TWO replies → Supervisor-Agent merge call
         Merged into single cohesive response
         Validation → OK
         No product grid (no product names mentioned)

Stage 10: Suggestions generated ("delivery" is primary_intent)
```

🤖 **Reply:**
```
Here's a full update on both your requests:

📦 **Delivery — Order ORD-99102**

2:00 PM and 3:00 PM today.

Your refund of **£14.99** (Reference: REF-77821) is currently being

```

```

---

**Turn 2**

💬 **User:** `"What about the cheese?"`

🔍 **Pipeline trace:**
```
Stage 4: History exists → LLM intent classification
         CONVERSATION HISTORY:
         ASSISTANT: "Your most recent order ORD-99102 contains: Whole Milk..."
         USER: "What about the cheese?"
         
         Intent LLM returns: "follow_up"

Stage 4b: _resolve_context("What about the cheese?", history)
          Context resolver sees previous reply mentioned "Cheddar × 1" in ORD-99102
          
          LLM returns:
          {
            "type": "resolved_query",
            "query": "Is the Cheddar cheese in order ORD-99102 included in any refund or promotion?"
          }
          
          message is replaced with resolved query for routing
          
Stage 7: "order" keyword → Order-Agent
Stage 8: Order-Agent reads context — no refund on ORD-99102
```

🤖 **Reply:**
```
The **Cheddar Cheese** in your order ORD-99102 is included in your
current delivery and is not subject to any active refund.

Your order is on its way — expected between 2:00 PM and 3:00 PM.
Did you have a concern about the cheese? I can raise a refund if needed.
```

💡 **Suggestions:** `Raise a refund for cheese` | `Check nutritional info` | `Track delivery` | `Contact support`

---

**Turn 3**

💬 **User:** `"Yes please"`

🔍 **Pipeline trace:**
```
Stage 4: History exists + "yes please" → check acknowledgements
         cleaned = "yes please"
         NOT in _ACKNOWLEDGEMENTS exact list
         → LLM intent check
         
         LLM sees context: assistant asked "Can I raise a refund if needed?"
         User said "Yes please"
         → Intent = "clarification_confirmation"

```

Stage 5: "calorie" in _RETAIL_KEYWORDS → domain = "retail"

Stage 6: ⚡ _search_db_for_product_question():
         "calories" IS in _PRODUCT_INFO_SIGNALS → activate
         Extract product terms: "bread", "sourdough" found in known_products list
         
         Query retail_chatbot.db (SQLite):
         SELECT * FROM inventory WHERE name LIKE '%sourdough%'
         
         Found: "Sainsbury's Sourdough Bread" — calorie=233kcal per 100g
         
         ✅ DB result found → BUILD CATALOG CARD → RETURN IMMEDIATELY
         ← ZERO Foundry agents called (huge latency saving)

## Example 10 — Voice Fast Path (Phone Call)

> **Scenario:** Jamie is calling on the phone. `is_voice=True`.

💬 **Caller says:** `"What is my refund status?"`

🔍 **Pipeline trace:**
```
Stage 1: Not a greeting → continue
Stage 2: Load customer data
Stage 3: Guardrail → ALLOWED

⚡ VOICE FAST PATH ACTIVATED (is_voice=True):
         Normal 10-stage pipeline is SKIPPED
         
         _classify_fallback("What is my refund status?")
         → "refund" keyword → agent_type = "delivery"... wait
         → "refund" keyword matched → agent_type = "refund" ✅
         
         _call_voice_openai(message, customer_data, history)
         
         Voice system prompt built:
         "You are a friendly Sainsbury's voice assistant on a phone call with Jamie.
          CUSTOMER DATA:
          - Name: Jamie
          - Email: jamie.thornton@example.co.uk
          - Nectar points: 3240
          - Latest order ORD-97830: Mature Cheddar Cheese, Greek Yoghurt.
            Status: Delivered. Total: £14.99.
            Refund REF-77821: £14.99 - Processing.
            Reason: Mouldy cheese.
          VOICE RULES: Reply in 1-2 short sentences. Under 25 words. No bullet points."
         
         GPT-4o call:
         - max_tokens=60    ← hard cap (forces short answer)
         - temperature=0.0  ← deterministic
         - Async call       ← sub-1s target
         
         ⏱️ Response time: ~0.4–0.8 seconds
         
         _static_suggestions("refund")  ← no LLM, instant lookup
```

🤖 **AI speaks:** (via Azure TTS `en-GB-SoniaNeural`)
```
"Your refund of fourteen pounds ninety-nine is currently being processed
and should arrive within three to five business days. Is there anything
else I can help you with?"
```

> **Total round-trip: ~0.6 seconds** (vs 3–8s for chat pipeline)

```

## Summary — Edge Case Coverage Map

| # | Edge Case | Path | Agents | Tools | LLM Calls |
|---|---|---|---|---|---|
| 1 | Simple greeting | Short-circuit | 0 | 0 | 0 |
| 2 | Single intent (delivery) | Direct keyword → 1 agent | 1 | 0 | 1 (suggestions) |
| 3 | Dual intent (delivery + refund) | Supervisor → 2 agents parallel | 3 | 1 | 3 |
| 4 | Triple intent (order+delivery+refund) | Supervisor → 3 agents parallel | 4 | 0 | 3 |
| 5 | Multi-turn follow-up | Intent: follow_up → context resolve → 1 agent | 1 | 1 | 3 |
| 6 | Ambiguous confirmation | Intent: clarification_confirmation → ask again | 0 | 0 | 2 |
| 7 | Out-of-context request | Guardrail BLOCKED | 0 | 0 | 1 (guardrail) |
| 8 | Nutrition question | DB shortcut (no agent) | 0 | 0 | 1 (suggestions) |
| 9 | Store hours + refund | Supervisor → 2 agents parallel | 3 | 1 | 3 |
| 10 | Voice call | Voice fast path | 0 | 0 | 1 (direct GPT-4o) |
| 11 | Hinglish query | Supervisor → order agent | 1 | 0 | 2 |
| 12 | Address update | Delivery agent + tool | 1 | 1 | 2 |
| 13 | Refund issue | Refund agent + tool | 1 | 1 | 2 |
| 14 | Stock check | Store agent + tool | 1 | 1 | 2 |
